# AGENTS.md

## Documentation Update Rule

When implementing or changing engine features, update documentation in the same change set.

Required updates:
1. `docs/DEVELOPMENT_GUIDE.ko.md`
2. `README.md` (feature list and examples when applicable)
3. At least one sample YAML (`public/sample/*.yaml` or `sample.yaml`) if DSL usage changed

If behavior changed but no DSL syntax changed, still update the guide's behavior notes and change log.
