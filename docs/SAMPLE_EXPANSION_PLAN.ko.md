# 샘플 장편화 설계안 (1차)

목표: 현재 데모를 "짧은 사건 재연"에서 "플레이어가 추리 흐름에 참여하는 장편형 비주얼노벨"로 확장한다.

## 1. 스토리라인 (기승전결)

- 기: 폭우로 고립된 료칸에서 계약 분쟁이 폭발한다.
- 승: 잠긴 다실에서 비명이 터지고, 밀실 살인으로 사건이 전환된다.
- 전: 알리바이 검증, 증거 충돌, 재현 실험을 통해 기존 진술이 무너진다.
- 결: 낚싯줄 트릭과 공범의 은폐 정황이 드러나며 범행 동기가 완성된다.

## 2. 장별 설계

1. `1.yaml` - 도입/사건 발생
- 인물 관계와 갈등 축 제시
- 컷신 + 사운드로 사건 발생 템포 고정

2. `2.yaml` - 초동 수사/첫 추리 게이트
- 용의자 진술 충돌
- `input`으로 사망 추정 시각 고정

3. `3.yaml` - 재현/증거 게이트
- 물증(찻잎, 젖지 않은 소매, 문틀 흠집) 연결
- `input`으로 핵심 물증 기억 고정

4. `4.yaml` - 범인 특정/에필로그
- 트릭 공개, 공범 정황, 사건 정리
- 감정 정리 + 후일담으로 여운 확보

## 3. 비주얼노벨 변환 규칙

- 씬당 목표를 하나만 둔다: 정보 제시 / 반전 / 결론.
- 감정 변화가 있는 대사 직전에는 `char.emotion` 교체를 먼저 배치한다.
- 클루 강조 구간은 `effect + sound + <speed=...>`를 함께 사용한다.
- 긴 설명 직전/직후 `wait`로 호흡을 확보한다.

## 4. 풍성화 장치 사용 매트릭스

- 컷신: 챕터 시작 리듬 형성
- 오버레이(`sticker`): 현장 통제감, 위기감 시각화
- 이펙트(`flash`, `darken`, `zoom`, `tilt`): 전환점 강조
- 입력 게이트(`input`): 플레이어 추론 참여
- 음악 전환: 도입(비), 수사(미스터리), 압박(텐션)

## 5. 분량 확장 기준

- 기존 2챕터 중심 구조를 4챕터 장편 구조로 확장
- 대사/행동을 "사건 제시 → 검증 → 재현 → 결론" 루프로 재배치
- 단서 회수를 위한 재언급 장면을 의도적으로 삽입
